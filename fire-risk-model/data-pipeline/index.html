    Data Pipeline¶

    The full data preparation workflow runs in four sequential steps.

    Step 1  Download Sentinel-2 patches      (Google Earth Engine)
Step 2  Tile patches to 256×256 px       (--pretile-npy)
Step 3  Align SCANFI v2 labels           (--align-scanfi)
Step 4  Split into train / val / test    (--split-data)

    Step 1 — Download Sentinel-2 patches¶

    Patches are downloaded from Google Earth Engine as June–August composites for a given year. Each patch is a 3-band GeoTIFF (Red, NIR, RedEdge) at 10 m resolution.

    Fire patches (burned areas)¶

    python scripts/earth_data.py \
    --region-geojson hotspots.geojson \
    --out-dir        data/patches \
    --year           2023

    Ecology patches (missing LCC classes)¶

    If SCANFI landcover classes 9–18 are absent from your training set, download additional reference patches sampled directly from the SCANFI LCC raster:

    python scripts/train_ecology.py --download-ecology \
    --input-dir      data/patches \
    --scanfi-dir     data/scanfi \
    --n-per-class    30 \
    --ecology-year   2021

    Sampling from SCANFI directly

    Ecology patches are sampled from the SCANFI nfiLandcover raster itself — not from a WorldCover proxy — so the downloaded patches are guaranteed to have the target LCC label when SCANFI alignment runs in Step 3.

    Step 2 — Tile patches¶

    Tile all *_bands.tif files into 256×256 px crops stored as .npy arrays:

    python scripts/train_ecology.py --pretile-npy \
    --input-dir  data/patches \
    --data-dir   data/tiles \
    --pipe-workers 4
    Flag Default Description
    --tile-size 256 Output tile size in pixels
    --pipe-workers 1 Parallel tiling processes

    Step 3 — Align SCANFI v2 labels¶

    Reprojects each SCANFI v2 layer to match every tile's CRS and extent, saving a *_scanfi.npy (9 channels, float16) alongside each *_bands.npy.

    python scripts/train_ecology.py --align-scanfi \
    --data-dir   data/tiles \
    --scanfi-dir data/scanfi \
    --pipe-workers 4

    SCANFI output channels¶

    Ch Name Type Description
    0 LCC Categorical Landcover class index ÷ 20
    1 TSD Continuous Forest age normalised to [0, 1] (÷ 200 yr)
    2 CC Continuous Crown closure [0, 1] (÷ 100)
    3 Pjack Continuous Jack pine fraction [0, 1]
    4 Pblack Continuous Black spruce fraction [0, 1]
    5 Balsam Continuous Balsam fir fraction [0, 1]
    6 Poplar Continuous Broadleaf fraction [0, 1]
    7 Spruce Continuous White/red pine fraction [0, 1]
    8 SpDiv Continuous Shannon species diversity [0, 1]

    Download caching¶

    By default, --align-scanfi downloads each of the 9 SCANFI layers once as a cropped local GeoTIFF covering your full tile region, then reprojects from that local file for every tile. This reduces HTTP requests from 9 × N_tiles to 9 total.

    # Cached downloads use --pipe-workers threads for the 9 layers
python scripts/train_ecology.py --align-scanfi \
    --data-dir     data/tiles \
    --pipe-workers 4          # 4 layers download in parallel

    SCANFI time steps

    Available time steps: 1985, 1990, 1995, 2000, 2005, 2010, 2015, 2020, 2025. The pipeline automatically selects the largest time step ≤ the patch acquisition year, so a 2022 fire patch is paired with SCANFI 2020 (the pre-fire snapshot).

    Step 4 — Split data¶

    Partition tiles into train / val / test sets with no spatial leakage (splits by source patch, not by individual tile):

    python scripts/train_ecology.py --split-data \
    --data-dir data/tiles

    Default split: 70 % train · 15 % val · 15 % test.

    LCC class coverage¶

    After alignment, check the LCC weight printout at the start of training. All 20 classes should be represented. Classes 9–18 (shrubland, grassland, cropland, developed, barren, water, snow, cloud, alpine, rock) require ecology patches downloaded in Step 1.

    LCC class weights:
  Class  1 (Conifer closed)  : weight = 0.92
  Class  2 (Conifer open)    : weight = 1.14
  ...
  Class 10 (Shrubland)       : weight = 2.31   ← needs ecology patches
  Class 15 (Water)           : weight = 1.87
  ...
