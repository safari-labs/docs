    1. Critical Fix Completed: Domain Layer Recovery ¶

    Problem¶

    The entire domain layer (server2/domain/) had no .py source files — only compiled .pyc bytecode in __pycache__/. This made the app completely non-functional (ModuleNotFoundError on startup).

    Fix Applied¶

    All 8 domain source files were reconstructed from .pyc bytecode analysis:

    File Lines Purpose
    domain/__init__.py 55 Re-exports all domain objects
    domain/types.py 133 Dataclasses: RFBudgetParams, LinkBudgetResult, FootprintResult, GroundStation, TelemetryPoint, LinkStatus
    domain/rf_math.py 82 Pure RF math: FSPL, EIRP, atmospheric loss, margin
    domain/coordinate_math.py 150 Geodetic transforms: Haversine, ENU, body-frame, quaternion rotation
    domain/antenna_registry.py 203 Pluggable antenna patterns: dipole, yagi, patch, helix, omni
    domain/rf_effects.py 195 Effect chain: multipath fading, pendulum swing, body shadowing, IMU variation
    domain/link_budget.py 176 Vectorized link budget computation
    domain/footprint.py 131 Ground power-grid computation

    Verification: App now starts, all 23 API routes register, all domain imports pass.

    2.2 Anti-Patterns Found¶

    P1 — CRITICAL: Massive Duplication Between Models and Domain¶

    Concept In models/ In domain/ Status
    Antenna patterns antenna_model.py (170 lines) antenna_registry.py (203 lines) Both exist, both register patterns
    Footprint grid footprint_model.py (150 lines) footprint.py (131 lines) footprint_model.py is DEAD CODE
    RF params rf_budget_model.py (plain dict) types.py (RFBudgetParams dataclass) Two sources of truth
    Coordinate math footprint_model.py (rotation matrix) coordinate_math.py Model version is DEAD

    Impact: antenna_controller.py does DUAL registration — it writes to both AntennaModel and AntennaRegistry on every custom antenna operation.

    P2 — Fat Route Handlers in app.py¶

    The /api/link_budget and /api/statistics routes each contain ~30 lines of parameter extraction that is nearly identical:

    # Repeated in BOTH routes:
attitude_jitter_deg = float(request.args.get("attitude_jitter_deg", 0))
enable_multipath_fading = request.args.get("enable_multipath_fading", "false").lower() == "true"
multipath_fade_depth_db = float(request.args.get("multipath_fade_depth_db", 0))
# ... 8 more parameters

    P3 — 3,784-Line Monolithic Frontend¶

    index2.html contains ALL frontend code: - ~200 lines CSS - ~600 lines HTML layout - ~2,900 lines JavaScript including: - Cesium 3D globe rendering - Plotly chart rendering - State management (20+ global variables) - API communication - PDF report generation (~300 lines) - CSV/JSON export - Custom antenna management - Playback loop with sub-frame interpolation - Auto-refresh with debounced listeners

    No separation of concerns whatsoever.

    P4 — Missing Static Files¶

    index2.html loads: 

    <script type="module" src="/static/js/state.js"></script>
<script type="module" src="/static/js/api.js"></script>
    But server2/static/js/ directory is empty — these files don't exist.

    P5 — Dead Code¶

    File Status
    services/shared_service.py Explicitly marked DEPRECATED, imported by nothing
    models/footprint_model.py Superseded by domain/footprint.py
    Most of models/antenna_model.py Superseded by domain/antenna_registry.py

    P6 — Hardcoded Ground Station in Multiple Places¶

    Ground station coordinates appear in: 1. app.py line 47: GROUND_STATION = {"lat": 48.5678, ...} 2. domain/types.py: GroundStation(lat=48.5678, ...) 3. config.py line ~100: GROUND_STATION_LAT = 48.5678

    Three separate definitions of the same constant.

DELETED:
  × models/antenna_model.py  → AntennaRegistry replaces it
  × models/footprint_model.py → domain/footprint.py replaces it
  × services/shared_service.py → was already dead code

KEPT (simplified):
  ✓ models/rf_budget_model.py → thin wrapper around RFBudgetParams
  ✓ models/telemetry_model.py → XLSX parser (no domain equivalent)

    Key Principles¶

    1. Single Source of Truth: RFBudgetParams is THE parameter object. RFBudgetModel becomes a thin persistence wrapper.
    2. Domain Purity: Domain layer already has zero Flask dependencies — preserve this.
    3. Thin Routes: app.py shrinks from 532 to ~200 lines by moving param extraction to controllers.
    4. No Duplication: Delete AntennaModel, FootprintModel, SharedService.

    4. Backend Refactor Plan ¶

    Phase 1: Eliminate Dead Code (Low Risk)¶

    1. Delete services/shared_service.py — unused
    2. Delete models/footprint_model.py — superseded by domain/footprint.py
    3. Update models/__init__.py — remove FootprintModel export
    4. Update controllers/footprint_controller.py — already uses domain layer, just remove FootprintModel dependency

    Phase 2: Consolidate Antenna Registration (Medium Risk)¶

    1. Delete pattern functions from models/antenna_model.py (keep only as a thin facade if needed for backward compat)
    2. Remove dual registration in antenna_controller.py — register ONLY in AntennaRegistry
    3. Make AntennaModel.CONFIGS read from AntennaRegistry.all_configs() instead of maintaining its own copy

    Phase 3: Unify RF Budget State (Medium Risk)¶

    Replace RFBudgetModel (dict-based) with RFBudgetParams (dataclass):

    # models/rf_budget_model.py — AFTER
class RFBudgetModel:
    def __init__(self):
        self._params = RFBudgetParams()

    def get_all(self) -> dict:
        return self._params.to_dict()

    def update(self, data: dict):
        self._params.update_from_dict(data)

    def get_params(self) -> RFBudgetParams:
        return self._params

    Phase 4: Extract Parameter Parsing from app.py (Low Risk)¶

    Move the 30-line parameter extraction blocks from /api/link_budget and /api/statistics into LinkBudgetController.parse_request():

    # BEFORE (in app.py, duplicated twice):
attitude_jitter_deg = float(request.args.get("attitude_jitter_deg", 0))
enable_multipath_fading = request.args.get("enable_multipath_fading", "false").lower() == "true"
# ... 8 more lines

# AFTER (in link_budget_controller.py):
@staticmethod
def parse_rf_overrides(args) -> dict:
    return {
        "attitude_jitter_deg": float(args.get("attitude_jitter_deg", 0)),
        "enable_multipath_fading": args.get("enable_multipath_fading", "false").lower() == "true",
        ...
    }

    Phase 5: Consolidate Ground Station (Low Risk)¶

    5. Frontend Refactor Plan ¶

    Current State: 3,784 Lines in One File¶

    The frontend is a monolith mixing: - Layout HTML - CSS styling - 20+ global state variables - Cesium 3D rendering (patterns, trajectory, beam, footprint) - Plotly charts (margin, RSSI, pattern, analysis) - API communication (10+ fetch calls) - PDF report generation (~300 lines) - Playback loop with rAF - Event handlers with debouncing

    Proposed Module Split¶

    Module Responsibility Est. Lines
    index.html Layout + CSS + script imports ~800
    static/js/state.js Single reactive state store, all global vars ~150
    static/js/api.js All fetch() calls, error handling ~200
    static/js/cesium.js Cesium viewer, patterns, trajectory, beam, footprint ~800
    static/js/charts.js Plotly charts: margin, RSSI, pattern, analysis ~400
    static/js/report.js PDF generation, CSV/JSON export ~400
    static/js/app.js Init, playback loop, event wiring, auto-refresh ~400

    Migration Strategy¶

    1. Create state.js — Extract all global variables into a single state object: 

      // state.js
export const state = {
  telemetry: [],
  budgetData: [],
  antennaConfigs: {},
  currentIdx: 0,
  layerVis: { trajectory: true, beam: true, ... },
  // ... all 20+ globals
};

    2. Create api.js — Extract all fetch calls: 

      // api.js
export async function loadBudget(tx, rx, params) { ... }
export async function loadStats(tx, rx, params) { ... }
export async function loadFootprint(params) { ... }

    3. Create remaining modules — Extract by DOM/rendering responsibility

    4. Update index.html — Replace inline <script> with ES module imports

    Key Decision: ES Modules vs Build Tool¶

    Recommended: ES Modules (no build step)

    The app already references <script type="module">. Native ES modules work in all modern browsers and keep the development workflow simple (no webpack/vite needed for a small team).

    6. Bug Fixes Analysis ¶

    BUG-1: Duplicate Event Handlers on Checkboxes (CONFIRMED)¶

    Location: index2.html lines 3501-3537 (setupAutoRefreshListeners())

    Problem: Checkbox elements have BOTH: - Inline onchange="applyRFBudget()" in HTML attributes - Programmatic addEventListener('change', ...) in setupAutoRefreshListeners()

    This causes applyRFBudget() to fire twice on every checkbox change.

    Affected elements: - #enable-multipath-fading - #enable-pendulum-swing - #enable-body-shadowing - #enable-imu-variation - #enable-attitude-rotation

    Fix: Remove the inline onchange attributes and rely solely on setupAutoRefreshListeners().

    BUG-2: Missing Static JS Files¶

    Location: index2.html lines ~20

    Problem: References static/js/state.js and static/js/api.js which don't exist. The <script> tags load empty modules that silently fail.

    Fix: Either create these files (part of frontend refactor) or remove the dead <script> tags.

    BUG-3: Frame Counter Pattern Update Miss¶

    Location: index2.html updateCesiumPatterns() function

    Problem: The function uses frameCounter % N to throttle updates. When settings change (e.g., antenna type) without an index change, the pattern update may be skipped until the next matching frame counter modulus.

    Fix: Add a settingsChanged flag that forces an immediate pattern update, then resets.

    BUG-4: Telemetry Overwrite on Budget Load¶

    Location: index2.html loadBudget() function

    Problem: loadBudget() replaces the telemetry array from the budget API response, but if the call fails, the old telemetry is preserved with stale _imuAccelerations cache.

    Fix: Clear _imuAccelerations cache regardless of success/failure, or better yet, compute it lazily from current telemetry.

    BUG-5: Double Telemetry Upload Endpoints¶

    Location: app.py lines 174-228

    Problem: Two nearly identical endpoints: - POST /api/telemetry/upload (returns {message, count})
    - POST /api/upload_telemetry (returns {ok, message})

    Different response formats for the same operation.

    Fix: Deprecate one, keep a single canonical endpoint.

    BUG-6: FootprintModel Imported But Unused¶

    Location: app.py line 14, models/__init__.py

    Problem: FootprintModel is imported and instantiated but the footprint controller actually calls domain.footprint.compute_footprint_grid. The model instance is passed to FootprintController.__init__ but may not be used.

    Fix: Remove after Phase 1 of backend refactor.

    7. Coupling Reduction Strategy ¶

    Current Coupling Hot Spots¶

    antenna_controller.py ──→ AntennaModel (legacy)
                      ──→ AntennaRegistry (domain)
                      ──→ AntennaDefinition (domain)

link_budget_controller.py ──→ AntennaModel
                          ──→ RFBudgetModel
                          ──→ TelemetryModel
                          ──→ RFBudgetParams (domain)
                          ──→ compute_link_budget (domain)

app.py ──→ ALL models (direct access)
       ──→ ALL controllers
       ──→ AntennaRegistry (for pattern helpers)
       ──→ request (Flask global)

    Reduction Plan¶

    Coupling Current Target
    antenna_controller → AntennaModel Direct REMOVE (use AntennaRegistry only)
    app.py → models Direct access (e.g., rf_budget_model.get() in routes) Via controllers only
    app.py → AntennaRegistry Direct import for pattern helpers Move helpers to AntennaController
    Ground station 3 separate definitions Single config.GROUND_STATION
    link_budget_controller → 3 models Constructor injection of 3 models Inject RFBudgetParams + telemetry list

    Adding New RF Effects (Already Plugin-Ready!)¶

    from server2.domain import RFEffectsChain, EffectContext
import numpy as np

def rain_attenuation(ctx: EffectContext, rain_rate_mm_hr: float = 10.0) -> np.ndarray:
    """Rain attenuation effect for frequencies > 10 GHz."""
    k = 0.01 * rain_rate_mm_hr ** 0.65  # ITU-R P.838
    return k * ctx.dist_km

# Add to chain
chain = RFEffectsChain()
chain.add(rain_attenuation, rain_rate_mm_hr=25.0)

    tx = request.args.get("tx", "dipole_half_wave")

    gs_lon = float(request.args.get("gs_lon", GROUND_STATION["lon"]))

    body_shadow_angle_range_deg = float(request.args.get("body_shadow_angle_range_deg", 45))

    budget, status = link_budget_controller.calculate_link_budget(
        tx=tx, rx=rx, gs_lat=gs_lat, gs_lon=gs_lon, gs_alt=gs_alt,

        enable_multipath_fading=enable_multipath_fading,
        # ... 6 more parameters

    if status == 200:
        return jsonify(budget.to_dict() if hasattr(budget, 'to_dict') else budget), 200
    return jsonify(budget), status

    @app.route("/api/link_budget")
def api_link_budget():
    result, status = link_budget_controller.handle_request(request.args)
    return jsonify(result), status
