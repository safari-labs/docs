    MVC Data Flow Diagrams¶

    1. Footprint Calculation Flow¶

    Browser Request
    │
    ├─→ /api/footprint?lat=48.5&lon=-81.3&alt=10000&pitch=10...
    │
    ↓
app.py (Flask Route)
    │
    ├─→ Extract parameters from request
    │
    ↓
FootprintController.calculate()
    │
    ├─→ Validate inputs
    ├─→ Fetch antenna configuration
    │
    ↓
FootprintModel.calculate_grid()
    │
    ├─→ CoordinateService.geodetic_to_enu()
    │   └─→ Transform balloon position to local coordinates
    │
    ├─→ AntennaService.yagi_pattern()
    │   └─→ Calculate antenna gain pattern
    │
    ├─→ RFService.free_space_path_loss()
    │   └─→ Calculate FSPL for each grid point
    │
    ├─→ NumPy vectorized calculation
    │   └─→ 25×25 grid with rotation matrix applied
    │
    ↓
RSSI Grid [[-120, -110, ...], [...]]
    │
    ├─→ Convert to JSON
    ├─→ Attach metadata (min, max, center)
    │
    ↓
Browser Response JSON
    │
    ├─→ JavaScript receives grid
    ├─→ rssiToRGBA() maps values to colors
    ├─→ Cesium.GroundPrimitive renders grid
    │
    ↓
Visual Update on Map

    2. Link Budget Calculation Flow¶

    Browser Request
    │
    ├─→ /api/link_budget?tx=yagi&rx=dipole_half_wave
    │
    ↓
app.py (Flask Route)
    │
    ↓
LinkBudgetController.calculate_link_budget()
    │
    ├─→ TelemetryModel.get_all()
    │   └─→ Load all balloon position/altitude points
    │
    ├─→ For each telemetry point:
    │
    ├─→ CoordinateService.distance_elevation_azimuth()
    │   └─→ Calculate distance, elevation, azimuth to ground station
    │
    ├─→ AntennaModel.get_gain()
    │   └─→ Get TX gain at elevation angle
    │   └─→ Get RX gain at azimuth
    │
    ├─→ RFService.free_space_path_loss()
    │   ├─→ Calculate FSPL
    │   └─→ Add atmospheric loss
    │
    ├─→ RFService.calculate_rssi()
    │   ├─→ EIRP - PathLoss - Losses + Gains
    │   └─→ Compare with RX sensitivity
    │
    ├─→ Determine link status
    │   └─→ NOMINAL / WARNING / MARGINAL / LINK_LOST
    │
    ↓
Array of link budgets (one per telemetry point)
    │
    ├─→ Merge with telemetry data
    ├─→ Calculate statistics (min/max/avg margin)
    │
    ↓
Browser Response JSON
    │
    ├─→ Plot link budget over time
    ├─→ Color-code regions (green/yellow/red)
    │
    ↓
Link Budget Visualization

    3. Telemetry Upload Flow¶

    Browser File Input
    │
    ├─→ Select XLSX file (tm_test.xlsx)
    │
    ↓
app.py (Flask Route)
    │
    ├─→ /api/telemetry/upload [POST]
    │
    ↓
TelemetryController.load_file()
    │
    ├─→ Save file to telemetry_uploads/
    │
    ↓
TelemetryModel.load_from_xlsx()
    │
    ├─→ openpyxl parses XLSX
    │
    ├─→ Extract columns:
    │   ├─→ Latitude (degrees)
    │   ├─→ Longitude (degrees)
    │   ├─→ Altitude (meters)
    │   ├─→ Pitch, Roll, Yaw (degrees)
    │   ├─→ Time (seconds)
    │
    ├─→ Store in self.data (list of dicts)
    │
    ↓
TelemetryModel.get_all()
    │
    └─→ Return data as JSON
        │
        ├─→ Browser receives telemetry
        ├─→ Initialize map at first point
        ├─→ Prepare for link budget calculation
        │
        ↓
Ready for simulation

    5. Request/Response Cycle¶

    ┌─────────────────────────────────────────────────────────┐
│  BROWSER REQUEST                                        │
│  GET /api/footprint?lat=48.5&lon=-81.3&alt=10000       │
└────────────────┬────────────────────────────────────────┘
                 │
                 ↓
        ┌────────────────┐
        │  Flask Route   │
        │  app.py        │
        └────────┬───────┘
                 │
                 ↓
        ┌────────────────────────┐
        │ FootprintController    │
        │ .calculate()           │
        │ - Validate params      │
        │ - Call model           │
        │ - Error handling       │
        └────────┬───────────────┘
                 │
                 ↓
        ┌────────────────────────┐
        │ FootprintModel         │
        │ .calculate_grid()      │
        │ - Vectorized calcs     │
        │ - Rotation matrix      │
        │ - RSSI per grid point  │
        └────────┬───────────────┘
                 │
                 ├─→ CoordinateService.geodetic_to_enu()
                 ├─→ AntennaService.yagi_pattern()
                 ├─→ RFService.free_space_path_loss()
                 │
                 ↓
        ┌────────────────────────┐
        │ Result Dict            │
        │ {                      │
        │   "grid": [...],       │
        │   "min": -120,         │
        │   "max": -50,          │
        │   "center": {...}      │
        │ }                      │
        └────────┬───────────────┘
                 │
                 ↓
        ┌────────────────────────┐
        │ JSON Response          │
        │ jsonify(result), 200   │
        └────────┬───────────────┘
                 │
                 ↓
┌──────────────────────────────────────────────────────────┐
│ BROWSER RESPONSE                                         │
│ 200 OK                                                   │
│ Content-Type: application/json                           │
│ {                                                        │
│   "grid": [[...], ...],                                 │
│   "min": -120,                                           │

         ↓

    6. Feature Isolation Example¶

    Adding a new antenna type¶

    Change: Only models/antenna_model.py

    antenna_patterns = {
    "dipole_half_wave": {...},
    "yagi": {...},
    "patch": {...},
    "helix": {...},
    "omni": {...},
    "NEW_ARRAY": {...}  # ← ADD HERE ONLY
}

    No impact on: - Controllers (still call antenna_model.get_gain()) - Services (still call antenna_patterns) - Routes (still call FootprintController.calculate()) - Frontend (still receives same response format) - Other features (telemetry, RF budget, link budget)

    Changing RF calculation¶

    Change: Only services/shared_service.py and models/rf_budget_model.py

    @staticmethod
def free_space_path_loss(freq_mhz: float, distance_m: float) -> float:
    # OLD: return 20 * np.log10(distance_m) + 20 * np.log10(freq_mhz) + 32.45
    # NEW: return 20 * np.log10(distance_m) + 20 * np.log10(freq_mhz) + 32.45 + atm_loss

    No impact on: - Antenna model (doesn't call this directly) - Controllers (call model, not service) - Telemetry (doesn't involve RF) - Other RF params (independent storage)

    Summary¶

    The MVC architecture achieves the design goal: - One feature = One controller + One (or few) models - Changes to Feature A → Don't affect Feature B - Clear data flow: Browser → Route → Controller → Model/Service - Easy testing: Each layer testable independently
