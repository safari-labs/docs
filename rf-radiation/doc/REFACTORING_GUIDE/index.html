    MVC Refactoring Guide¶

    What Was Done¶

    1. Code Reorganization¶

    Before (Monolithic): 

    server2/
├── server_2.py (973 lines - everything mixed)
└── index2.html

    After (MVC): 

    server2/
├── models/
│   ├── antenna_model.py      # Antenna data & patterns
│   ├── telemetry_model.py    # Telemetry file handling
│   ├── rf_budget_model.py    # RF parameters
│   └── footprint_model.py    # Footprint calculations
├── controllers/
│   ├── footprint_controller.py    # Footprint requests
│   ├── antenna_controller.py      # Antenna operations
│   ├── rf_budget_controller.py    # RF updates
│   ├── telemetry_controller.py    # File uploads
│   └── link_budget_controller.py  # Link calculations
├── services/
│   └── shared_service.py      # Coordinate & RF utilities
├── app.py                     # Flask routes (new)
└── server_2.py               # Legacy (kept for reference)

    2. Feature Isolation¶

    Each feature now has: - Model: Data representation (e.g., FootprintModel) - Controller: Business logic (e.g., FootprintController) - Service: Shared utilities (e.g., CoordinateService)

    Example: Footprint Feature 

    User Request → FootprintController.calculate()
  ↓
  ├─→ FootprintModel.calculate_grid()
  ├─→ AntennaModel.get_gain()
  ├─→ CoordinateService.*() 
  └─→ RFService.*()
  ↓
Returns RSSI grid JSON

    3. Dependency Flow¶

    app.py (Routes)
  ↓
Controllers (Business Logic)
  ↓
Models (Data) + Services (Utilities)

    Key Principle: No upward dependencies - Models don't import Controllers - Controllers don't know about Routes - Each component is testable in isolation

    How to Use the New Architecture¶

    Adding a New Feature¶

    Example: Adding a new RF calculation

    1. Create Model (models/new_feature_model.py): 

      class NewFeatureModel:
    def __init__(self):
        pass

    def calculate(self, params):
        # Core calculation logic
        return result

    2. Create Controller (controllers/new_feature_controller.py): 

      class NewFeatureController:
    def __init__(self, model):
        self.model = model

    def handle_request(self, params):
        result, status = self.model.calculate(params)
        return jsonify(result), status

    3. Add Route (app.py): 

      @app.route("/api/new_feature")
def api_new_feature():
    controller = NewFeatureController(model)
    return controller.handle_request(request.args)

    4. Register in __init__.py: 

      # controllers/__init__.py
from .new_feature_controller import NewFeatureController

__all__ = [..., 'NewFeatureController']

    Modifying Existing Code¶

    Change antenna pattern? → Edit models/antenna_model.py only Change RF calculation? → Edit services/shared_service.py only Change API response? → Edit controller method only Change input validation? → Edit controller method only

    No cascading changes! Other features won't be affected.

    Testing Strategy¶

    Unit Tests¶

    Test models in isolation: 

    from server2.models import AntennaModel

antenna = AntennaModel()
gain = antenna.get_gain("yagi", 30)
assert gain > 0

    Test services: 

    from server2.services import CoordinateService

east, north, up = CoordinateService.geodetic_to_enu(...)
assert isinstance(east, float)

    Integration Tests¶

    Test controllers with mocked models: 

    from server2.controllers import FootprintController
from unittest.mock import Mock

model = Mock()
model.calculate_grid.return_value = {...}
controller = FootprintController(model, antenna_model)
result, status = controller.calculate(...)
assert status == 200

    End-to-End Tests¶

    Test Flask routes: 

    curl "http://localhost:5050/api/footprint?lat=48.5&lon=-81.3..."

    Performance Considerations¶

    • Vectorized NumPy: All grid calculations use NumPy arrays (not Python loops)
    • Model Caching: Models cache computed values when possible
    • Lazy Loading: Telemetry loaded on-demand, not all at startup
    • Service Pooling: Could add pattern caching in AntennaService for repeated calculations

    Common Patterns¶

    Getting Data from a Model¶

    from server2.models import RFBudgetModel

budget = RFBudgetModel()
tx_power = budget.get("tx_power_dbm")

    Using Services¶

    from server2.services import CoordinateService, RFService

# Coordinate transformation
dist, elev, az = CoordinateService.distance_elevation_azimuth(...)

# RF calculation
fspl = RFService.free_space_path_loss(145.0, 100000)

    Creating a Controller¶

    class MyController:
    def __init__(self, model, antenna_model=None):
        self.model = model
        self.antenna_model = antenna_model

    def handle_request(self, params):
        try:
            result = self.model.calculate(params)
            return {"status": "ok", "data": result}, 200
        except Exception as e:
            return {"error": str(e)}, 400

    Troubleshooting¶

    Import Errors¶

    Problem: ModuleNotFoundError: No module named 'server2'

    Solution: 

    cd /path/to/server2
python3 main.py  # main.py handles path setup

    Missing Dependencies¶

    Problem: ImportError: No module named 'flask'

    Solution: 

    pip install flask flask-cors numpy openpyxl

    Controller Initialization Fails¶

    Problem: Controller can't find model

    Solution: Ensure model is imported and passed to controller constructor 

    from server2.models import FootprintModel
from server2.controllers import FootprintController

model = FootprintModel()
controller = FootprintController(model, antenna_model)
