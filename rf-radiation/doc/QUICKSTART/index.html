    Quick Start Guide¶

    5-Minute Setup¶

    Step 1: Install Dependencies¶

    pip install flask flask-cors numpy openpyxl

    Step 2: Start the Server¶

    cd /home/ouazo/Desktop/ASSFID/server2
python3 main.py

    Step 3: Open in Browser¶

    Visit: http://localhost:5050

    Basic Workflow¶

    1. Upload Telemetry¶

    1. Click "Upload Telemetry" button
    2. Select your XLSX file
    3. Wait for confirmation

    2. Configure RF Budget¶

    1. Open "RF Budget" panel
    2. Adjust TX power, RX sensitivity
    3. Select frequencies
    4. Click "Apply"

    3. Select Antennas¶

    1. Choose TX antenna type
    2. Choose RX antenna type
    3. View gains automatically calculated

    4. View Link Budget¶

    1. Click "Link Budget" tab
    2. See coverage along balloon path
    3. Check margin indicators (green/yellow/red)

    5. View Footprint¶

    1. Click "Footprint" tab
    2. See power grid visualization
    3. Adjust pitch/roll/yaw to rotate antenna
    4. Grid updates in real-time

    Common Tasks¶

    Upload New Telemetry File¶

    Web Interface: 

    Home → Upload Telemetry → Choose File → Submit

    API: 

    curl -F "file=@tm_test.xlsx" http://localhost:5050/api/telemetry/upload

    Get Link Budget Data¶

    API: 

    curl "http://localhost:5050/api/link_budget?tx=yagi&rx=dipole_half_wave"

    Response: Array of link budgets (one per telemetry point)

    Calculate Footprint at Specific Time¶

    API: 

    curl "http://localhost:5050/api/footprint?lat=48.5&lon=-81.3&alt=10000&pitch=10&roll=5&yaw=45&tx=yagi"

    Response: 25×25 RSSI grid in dBm

    Reset RF Budget to Defaults¶

    Web Interface: 

    RF Budget → Reset Button

    API: 

    curl -X POST http://localhost:5050/api/rf_budget/reset

    Get All Configuration¶

    API: 

    curl http://localhost:5050/api/config

    Troubleshooting¶

    Server won't start¶

    # Check Python version
python3 --version  # Need 3.7+

# Check dependencies
pip list | grep -i flask

# Install missing packages
pip install -r requirements.txt

    Port 5050 in use¶

    # Kill existing process
killall python3

# Or use different port (edit main.py)

    Telemetry file won't upload¶

    # Check file format
# - Must be .xlsx (Excel format)
# - Must have columns: Latitude, Longitude, Altitude, Pitch, Roll, Yaw

# Check file size
# - Max 10,000 rows recommended

    Link budget shows all "LINK_LOST"¶

    # Check RF Budget settings
# - TX Power too low?
# - RX Sensitivity set incorrectly?
# - Frequency correct?

# Reset to defaults:
# RF Budget → Reset Button

    Footprint visualization not updating¶

    # Try refreshing page (Ctrl+R)
# Check browser console (F12) for errors
# Verify antenna type is valid

    Key Files Explained¶

    main.py¶

    Entry point that starts the Flask server. Run this to start the application.

    config.py¶

    Centralized configuration. All settings in one place - RF budget defaults, antenna configs, ground station location, etc.

    server2/app.py¶

    Flask application with all API routes. Maps HTTP requests to controllers.

    server2/models/¶

    Data management. Each model handles one data type (antenna, telemetry, RF budget, footprint).

    server2/controllers/¶

    Business logic. Each controller orchestrates requests and calls models/services.

    server2/services/¶

    Shared utilities. Coordinate transformations, RF calculations, antenna patterns.

    server2/index2.html¶

    Web interface. HTML/CSS/JavaScript frontend running in browser.

    API Endpoints Quick Reference¶

    Endpoint Method Purpose
    /api/footprint GET Calculate power grid
    /api/link_budget GET Calculate link budget
    /api/antennas GET List antenna types
    /api/rf_budget GET Get RF parameters
    /api/rf_budget POST Update RF parameters
    /api/telemetry GET Get all telemetry
    /api/telemetry/<idx> GET Get single point
    /api/telemetry/upload POST Upload file
    /api/statistics GET Get link budget stats
    /api/config GET Get all config

    Performance Tips¶

    1. Large Telemetry Files: Upload via web interface (batches requests)
    2. Real-time Updates: Adjust "Grid Refresh Rate" slider (lower = faster)
    3. Antenna Patterns: Service caches patterns automatically
    4. Grid Calculations: Vectorized NumPy - handles 25×25 grids in <100ms

    Example Scenarios¶

    Scenario 1: Testing TX Power¶

    1. Upload balloon trajectory
    2. Set TX power to 10 dBm
    3. View link budget - note margin
    4. Change TX power to 20 dBm
    5. Link budget updates - higher margin everywhere

    Scenario 2: Antenna Comparison¶

    1. Set TX antenna to "dipole_half_wave"
    2. Note coverage statistics
    3. Switch to "yagi" antenna
    4. Margin improved due to higher gain

    Scenario 3: Real-time Antenna Pointing¶

    1. Open Footprint visualization
    2. Adjust pitch/roll/yaw sliders
    3. Watch power grid rotate on map
    4. Find optimal antenna pointing angle

    Scenario 4: Finding Blackout Zones¶

    1. Upload trajectory with high altitude variation
    2. Open Link Budget view
    3. Red zones show where signal is lost
    4. Yellow zones show marginal coverage
