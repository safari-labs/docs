    RF Link Budget Simulator v2¶

    A Flask-based RF link budget and footprint mapping tool for high-altitude balloon (HAB) projects. Features real-time power grid visualization in Cesium.js with antenna orientation support.

    Quick Start¶

    Installation¶

    # Clone the repository
cd /home/ouazo/Desktop/ASSFID/server2

# Install dependencies
pip install flask flask-cors numpy openpyxl

# Start the server
python3 main.py

    Server runs on http://localhost:5050

    Using the Application¶

    1. Open in browser: Visit http://localhost:5050
    2. Upload telemetry: Load XLSX file with balloon trajectory
    3. Configure RF Budget: Adjust TX power, RX sensitivity, frequencies
    4. Select Antennas: Choose TX/RX antenna types
    5. View Link Budget: Check coverage statistics
    6. View Footprint: See power grid visualization with antenna orientation

    Architecture¶

    This project uses Model-View-Controller (MVC) pattern:

    • Models: Data management (antenna configs, telemetry, RF params, footprint calculations)
    • Controllers: Business logic and request handling
    • Services: Shared utilities (coordinate transforms, RF calculations)
    • Views: Flask routes and HTML frontend

    See MVC_ARCHITECTURE.md for detailed architecture docs. See REFACTORING_GUIDE.md for migration details.

    Features¶

    1. Footprint Mapping¶

    • Real-time power grid calculation around balloon position
    • Antenna orientation support (Pitch/Roll/Yaw)
    • Vectorized NumPy calculations for performance
    • Cesium.js visualization with color gradient (Blue→Red)

    2. Link Budget Analysis¶

    • TX↔RX link analysis for all telemetry points
    • Multiple antenna patterns (dipole, yagi, patch, helix, omni)
    • Distance, elevation, azimuth calculations
    • Link margin indicators (NOMINAL/WARNING/MARGINAL/LINK_LOST)

    3. RF Budget Management¶

    • TX power (0-30 dBm)
    • RX sensitivity (-110 to -100 dBm)
    • Frequency configuration (VHF/UHF support)
    • Cable losses, atmospheric absorption
    • Realism effects (Fresnel zones, multipath, antenna mismatch)

    4. Telemetry Processing¶

    • XLSX file upload and parsing
    • Attitude angles (Pitch/Roll/Yaw) support
    • Altitude and position tracking
    • Real-time data updates

    • Python 3.7+

    • NumPy: Vectorized calculations
    • openpyxl: XLSX file parsing
    • Cesium.js: 3D mapping (frontend)
    • Plotly.js: Charts (frontend)

    Configuration¶

    Ground Station Position¶

    Edit in app.py: 

    GROUND_STATION = {"lat": 48.5678, "lon": -81.3655, "alt": 285.8}

    Telemetry Path¶

    UPLOAD_DIR = os.path.join(os.path.dirname(__file__), '..', 'telemetry_uploads')

    Usage Examples¶

    Python API¶

    from server2.models import AntennaModel, FootprintModel, RFBudgetModel
from server2.controllers import FootprintController

# Initialize
antenna = AntennaModel()
footprint = FootprintModel()
rf = RFBudgetModel()

# Calculate footprint
controller = FootprintController(footprint, antenna)
result, status = controller.calculate(
    lat=48.5, lon=-81.3, alt=10000,
    pitch=10, roll=5, yaw=45,
    frequency_mhz=145.0,
    tx="yagi",
    grid_n=25,
    rf_budget=rf.get_all()
)

    Frontend JavaScript¶

    // Fetch footprint
fetch('/api/footprint?lat=48.5&lon=-81.3&alt=10000&pitch=10&roll=5&yaw=45')
  .then(r => r.json())
  .then(data => updateFootprint(data))

// Upload telemetry
const formData = new FormData()
formData.append('file', fileInput.files[0])
fetch('/api/telemetry/upload', {method: 'POST', body: formData})
  .then(r => r.json())
  .then(data => console.log(data))

    Troubleshooting¶

    Port 5050 Already in Use¶

    # Find process using port 5050
lsof -i :5050

# Kill the process
kill -9 <PID>

# Or use a different port (edit main.py)

    Import Errors¶

    # Ensure dependencies installed
pip install -r requirements.txt

# Or manually
pip install flask flask-cors numpy openpyxl

    No Telemetry Data¶

    # Upload file via web interface
# Or place tm_test.xlsx in project root
python3 main.py

    Known Issues¶

    1. Mesh doesn't follow TX footprint - Antenna orientation visualization not updating correctly
    2. Status: Under investigation

    3. Workaround: Manually refresh page to see updated grid

    4. Large telemetry files (>10k points) - Slow link budget calculation

    5. Mitigation: Vectorized NumPy should handle 1k-5k points efficiently
    6. Future: Add caching and batching

    Performance Notes¶

    • Footprint grid (25×25): ~50-100ms calculation
    • Link budget (1000 points): ~200-300ms
    • Frontend visualization: 60 FPS with throttling

    Code Style¶

    • PEP 8 compliance
    • Type hints where applicable
    • Docstrings for all functions
    • No external dependencies unless necessary

    Contributing¶

    When adding features: 1. Keep models independent (no controller imports) 2. Centralize shared logic in services/ 3. Document API changes 4. Test in isolation before integration
