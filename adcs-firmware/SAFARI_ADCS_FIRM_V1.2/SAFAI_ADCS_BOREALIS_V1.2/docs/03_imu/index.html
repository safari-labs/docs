    Subsystem — IMU ICM-20948¶

    Sensor: ICM-20948 — TDK InvenSense (Adafruit #4554)
    Axes: Accelerometer (3) + Gyroscope (3) + Magnetometer AK09916 (3) = 9-axis
    Bus: I2C1, 400 kHz, address 0x68
    Files: include/imu.h · src/imu.cpp

    1. Hardware Connections¶

    @startuml
skinparam componentStyle rectangle

node "Raspberry Pi Pico (RP2040)" {
    component "GPIO6  (I2C1 SDA)" as SDA
    component "GPIO7  (I2C1 SCL)" as SCL
    component "GPIO12 (AD0)"      as AD0
}

component "ICM-20948\n(I2C addr 0x68)" as ICM {
    component "Accel\n±4 g" as ACC
    component "Gyro\n±500 dps" as GYR
    component "Mag AK09916\n10 Hz" as MAG
}

SDA <--> ICM : SDA
SCL  --> ICM : SCL
AD0  --> ICM : AD0 = LOW → addr 0x68

note bottom of AD0
  Held LOW at startup.
  Address locked to 0x68,
  never changes at runtime.
end note
@enduml
    GPIO Role Direction
    GPIO6 I2C1 SDA bidirectional
    GPIO7 I2C1 SCL output (master)
    GPIO12 AD0 address select OUTPUT, always LOW

    Implementation note: Wire1 is not exported by the Arduino-Mbed variant for the Pico.
    A dedicated instance is created: static TwoWire _wire1(PIN_IMU_SDA, PIN_IMU_SCL);

    3. Initialization Sequence — init()¶

    @startuml
participant ImuModule   as imu
participant TwoWire     as wire
participant "Adafruit\nICM20948" as lib
participant "Arduino\nGPIO"     as gpio

imu  -> gpio : pinMode(GPIO12, OUTPUT)
imu  -> gpio : digitalWrite(GPIO12, LOW)  ' AD0 = LOW → address 0x68
imu  -> wire : _wire1.begin()
imu  -> wire : _wire1.setClock(400000)

imu  -> lib  : begin_I2C(0x68, &_wire1)
alt sensor found
    lib --> imu : true
    imu -> lib  : setAccelRange(±4 g)
    imu -> lib  : setGyroRange(±500 dps)
    imu -> lib  : setMagDataRate(10 Hz)
    imu -> imu  : _ready = true\n_error = OK
    imu -> imu  : Serial.println("[IMU] OK")
else not found
    lib --> imu : false
    imu -> imu  : _error = NOT_FOUND\n_ready = false
    imu -> imu  : Serial.println("[IMU] not found")
end
@enduml

    4. Measurement Sequence — read()¶

    @startuml
participant ImuModule        as imu
participant "Adafruit\nICM20948" as lib
participant "g_adcs.imu"     as data

imu -> lib  : getEvent(&accel, &gyro, &temp, &mag)
note right of lib
  I2C1 burst read (400 kHz):
  · 14 bytes accel+gyro+temp
  · 8 bytes mag (AK09916)
  · float conversion ×9 axes
  NOP library ≈ 76 ops
end note
lib --> imu : sensors_event_t filled

imu -> imu  : amag² = ax²+ay²+az²

alt amag² == 0.0  [failed read sentinel]
    imu -> imu  : _error = DATA_INVALID\n_failCount++
    imu -> data : valid = false
    imu --> imu : return false

    alt _failCount >= 3
        imu -> imu : ' watchdog notified via sendStatus() in main
    end

else valid data
    imu -> imu  : _failCount = 0\n_error = OK
    imu -> data : ax/ay/az = accel  [m/s²]
    imu -> data : gx/gy/gz = gyro   [rad/s]
    imu -> data : mx/my/mz = mag    [µT]
    imu -> data : valid = true
    imu --> imu : return true
end
@enduml

    5. Sensor Configuration¶

    Parameter Value Register constant
    Accelerometer range ±4 g ICM20948_ACCEL_RANGE_4_G
    Gyroscope range ±500 dps ICM20948_GYRO_RANGE_500_DPS
    Magnetometer rate 10 Hz AK09916_MAG_DATARATE_10_HZ
    I2C address 0x68 IMU_I2C_ADDR (config.h)
    I2C frequency 400 kHz IMU_I2C_FREQ (config.h)

    6. Output — g_adcs.imu¶

    Field Unit Source axis
    ax, ay, az m/s² ICM-20948 accelerometer
    gx, gy, gz rad/s ICM-20948 gyroscope
    mx, my, mz µT AK09916 magnetometer
    valid bool false if read fails or amag² = 0

    7. Attitude Fusion (future)¶

    @startuml
component "Gyroscope\n(rad/s)" as GYR
component "Accelerometer\n(m/s²)" as ACC
component "Magnetometer\n(µT)" as MAG
component "Fusion Algorithm\n(TBD)" as FUSE
component "Quaternion\nq = [w, x, y, z]" as QUAT

GYR --> FUSE : angular rate\n(high-freq propagation)
ACC --> FUSE : gravity vector\n(pitch/roll correction)
MAG --> FUSE : Earth field\n(yaw correction)
FUSE --> QUAT

note bottom of FUSE
  Algorithm TBD after sensor characterization:
  · Complementary filter
  · Madgwick
  · Mahony
  · EKF
end note
@enduml

    Quaternion avoids gimbal lock inherent to Euler angles.
