    Subsystem — External Watchdog¶

    Role: External watchdog module — monitors ADCS health, resets RP2040 via RUN pin on timeout
    Interface: I2C0 (GPIO4/5) + CLK output (GPIO19)
    I2C address: 0x55 (watchdog side — confirm before deployment)
    Files: include/watchdog.h · src/watchdog.cpp

    1. Hardware Connections¶

    @startuml
skinparam componentStyle rectangle

node "Raspberry Pi Pico (RP2040)" {
    component "GPIO4  (I2C0 SDA)" as SDA
    component "GPIO5  (I2C0 SCL)" as SCL
    component "GPIO19 (CLK OUT)"  as CLK
    component "RUN    (reset IN)" as RUN
}

component "External Watchdog\nModule (0x55)" as WD {
    component "I2C Slave\n(receives status)" as I2C_S
    component "CLK counter\n(timeout detect)"  as CTR
    component "Reset driver\n(open-drain)"      as RST
}

SDA <--> I2C_S : SDA (I2C0)
SCL  --> I2C_S : SCL (I2C0)
CLK  --> CTR   : CLK pulse (1 kHz)\nidles LOW
RST  --> RUN   : RESET (active LOW)

note bottom of RST
  If CLK stops or status byte
  indicates sensor failure,
  watchdog pulls RUN LOW
  to reset the RP2040.
end note
@enduml
    GPIO Role Direction State
    GPIO4 I2C0 SDA bidirectional managed by Wire (default Mbed Pico pins)
    GPIO5 I2C0 SCL output managed by Wire
    GPIO19 CLK output OUTPUT idles LOW; pulsed once per read()
    RUN hardware reset INPUT pulled LOW by watchdog on timeout

    3. Initialization Sequence — init()¶

    @startuml
participant WatchdogComm as wd
participant "Arduino\nGPIO"  as gpio
participant "Wire\n(I2C0)"   as wire

wd   -> gpio : pinMode(GPIO19, OUTPUT)
wd   -> gpio : digitalWrite(GPIO19, LOW)  ' CLK idles LOW
wd   -> wire : Wire.begin()
note right : I2C0 on GPIO4/5\n(Mbed Pico default — no pin remapping needed)

wd   -> wire : Wire.beginTransmission(0x55)
wd   -> wire : Wire.endTransmission() → err

alt err == 0  [watchdog responds]
    wd -> wd   : _ready = true\n_error = OK
    wd -> wd   : g_adcs.watchdog.connected = true
    wd -> wd   : Serial.println("[Watchdog] OK")
else err != 0  [NACK — absent or powered off]
    wd -> wd   : _error = NOT_FOUND\n_ready = false
    wd -> wd   : g_adcs.watchdog.connected = false
    wd -> wd   : Serial.println("[Watchdog] not found")
    note right : Non-fatal — other modules\ncontinue normally
end
@enduml

    4. Per-cycle Sequence — read() + sendStatus()¶

    @startuml
participant "main.cpp"       as main
participant WatchdogComm     as wd
participant "Wire (I2C0)"    as wire
participant "GPIO19 (CLK)"   as clk
participant "g_adcs.watchdog" as data

main -> wd   : read()

wd   -> wd   : tickClock()
wd   -> clk  : digitalWrite(HIGH)
wd   -> wd   : delayMicroseconds(500)
wd   -> clk  : digitalWrite(LOW)
wd   -> wd   : delayMicroseconds(500)
note right : 1 full CLK pulse\nperiod = 1000 µs = 1 kHz

wd   -> wire : Wire.beginTransmission(0x55)
wd   -> wire : Wire.endTransmission() → err

alt err == 0
    wd   -> data : connected = true
    wd   --> wd  : return true
else err != 0
    wd   -> data : connected = false
    wd   -> wd   : _error = NOT_FOUND
    wd   --> wd  : return false
end

main -> wd   : sendStatus(status_byte)
wd   -> wire : Wire.beginTransmission(0x55)
wd   -> wire : Wire.write(status_byte)
wd   -> wire : Wire.endTransmission()
@enduml

    5. Status Byte — sendStatus()¶

    @startuml
start

:build status_byte = 0;

if (g_adcs.imu.valid?) then (yes)
    :status_byte |= 0x01  [bit 0 IMU_OK];
endif

if (g_adcs.altimeter.valid?) then (yes)
    :status_byte |= 0x02  [bit 1 ALT_OK];
endif

if (g_adcs.pt1000.active_count > 0?) then (yes)
    :status_byte |= 0x04  [bit 2 PT_OK];
endif

:WatchdogComm::sendStatus(status_byte)
I2C write → 0x55;

stop
@enduml
    Bit Mask Meaning
    0 0x01 (WD_STATUS_IMU_OK) g_adcs.imu.valid == true
    1 0x02 (WD_STATUS_ALT_OK) g_adcs.altimeter.valid == true
    2 0x04 (WD_STATUS_PT_OK) g_adcs.pt1000.active_count > 0
    3–7 — reserved

    6. CLK Signal Timing¶

    @startuml
title GPIO19 CLK — WD_CLK_HALF_US = 500 µs · period = 1 ms · one pulse per loop() cycle (1 s)

robust "GPIO19 CLK" as CLK

@0
CLK is LOW

@1000
CLK is HIGH

@1500
CLK is LOW

@2500
CLK is HIGH

@3000
CLK is LOW

@3000
note top of CLK : one pulse = tickClock()
@enduml

    The CLK pin idles LOW. One pulse is sent at the start of each read() call. The watchdog uses the absence of this pulse (over its timeout period) to trigger a reset.
