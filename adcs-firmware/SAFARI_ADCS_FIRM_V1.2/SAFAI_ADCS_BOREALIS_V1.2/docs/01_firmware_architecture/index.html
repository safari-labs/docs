    Firmware Architecture — SAFARI ADCS V1.2¶

    Platform: Raspberry Pi Pico (RP2040) · Arduino-Mbed · PlatformIO
    Revision: V2.1 — 2026-06-10

    1. Overview¶

    The firmware is organized around a central SensorManager that orchestrates up to four independent sensor modules. Each module implements the same abstract interface (SensorModule). Results are written to a single shared data bus (AdcsData g_adcs). No module depends on another.

    2. Class Hierarchy¶

    @startuml
skinparam classAttributeIconSize 0
skinparam classFontSize 13

abstract class SensorModule {
    + {abstract} init()    : bool
    + {abstract} read()    : bool
    + {abstract} isReady() : bool
    + {abstract} name()    : const char*
    + {abstract} lastError(): SensorError
}

enum SensorError {
    OK
    NOT_FOUND
    BAD_ID
    DATA_INVALID
    READ_FAIL
}

class ImuModule {
    - _icm       : Adafruit_ICM20948
    - _wire1     : TwoWire
    - _ready     : bool
    - _failCount : uint8_t
    - _error     : SensorError
    + init()  : bool
    + read()  : bool
}

class AltimeterModule {
    - _sensor    : MS5607
    - _ready     : bool
    - _failCount : uint8_t
    - _error     : SensorError
    + init()  : bool
    + read()  : bool
}

class Pt1000Module {
    - _ready : bool
    - _error : SensorError
    + init()          : bool
    + read()          : bool
    - selectChannel(ch: uint8_t)
    - lutLookup(v: float) : float
}

class WatchdogComm {
    - _ready : bool
    - _error : SensorError
    + init()              : bool
    + read()              : bool
    + sendStatus(byte: uint8_t)
    - tickClock()
}

class SensorManager {
    - modules[MAX_MODULES] : SensorModule*
    - count                : uint8_t
    + registerModule(m: SensorModule*)
    + initAll()
    + readAll()
}

struct AdcsData {
    + imu        : {ax,ay,az · gx,gy,gz · mx,my,mz · valid}
    + altimeter  : {pressure · temperature · altitude · valid}
    + pt1000     : {temps[8] · valid[8] · active_count}
    + watchdog   : {connected}
}

SensorModule  <|--  ImuModule
SensorModule  <|--  AltimeterModule
SensorModule  <|--  Pt1000Module
SensorModule  <|--  WatchdogComm

SensorModule  ..>   SensorError
SensorManager  o--  SensorModule : "manages [0..4]"
SensorModule  ..>   AdcsData     : "writes own section"

note bottom of AdcsData : "extern AdcsData g_adcs\ndefined in sensors.cpp"
@enduml

    3. Shared Data Bus — AdcsData¶

    @startuml
skinparam objectAttributeIconSize 0

object "g_adcs : AdcsData" as g {
    imu.ax / ay / az    : float   ' m/s²
    imu.gx / gy / gz    : float   ' rad/s
    imu.mx / my / mz    : float   ' µT
    imu.valid           : bool
    ──────────────────────────────
    altimeter.pressure  : float   ' hPa
    altimeter.temperature: float  ' °C
    altimeter.altitude  : float   ' m
    altimeter.valid     : bool
    ──────────────────────────────
    pt1000.temps[8]     : float   ' °C
    pt1000.valid[8]     : bool
    pt1000.active_count : uint8_t
    ──────────────────────────────
    watchdog.connected  : bool
}
@enduml

    Each module writes only its own section. main.cpp reads all sections after readAll().

    4. Initialization Sequence — setup()¶

    @startuml
participant "main.cpp"      as main
participant SensorManager   as mgr
participant ImuModule       as imu
participant AltimeterModule as alt
participant Pt1000Module    as pt
participant WatchdogComm    as wd

main -> main : Serial.begin(115200)\nwait USB ≤ 3 s

main -> mgr : registerModule(&imu)
main -> mgr : registerModule(&altimeter)
main -> mgr : registerModule(&pt1000)
main -> mgr : registerModule(&watchdog)

main -> mgr : initAll()

mgr -> imu  : init()
imu --> mgr : true / false

mgr -> alt  : init()
alt --> mgr : true / false

mgr -> pt   : init()
pt  --> mgr : true / false

mgr -> wd   : init()
wd  --> mgr : true / false

note right of mgr
  Modules returning false
  are marked not-ready.
  readAll() will skip them.
end note
@enduml

    5. Main Loop Sequence — loop() (period = 1000 ms)¶

    @startuml
participant "main.cpp"      as main
participant SensorManager   as mgr
participant ImuModule       as imu
participant AltimeterModule as alt
participant Pt1000Module    as pt
participant WatchdogComm    as wd
participant "g_adcs"        as data

loop every 1000 ms  [paced via millis()]

    main -> mgr : readAll()

    mgr -> imu : read()
    imu -> data : write imu section
    imu --> mgr : true / false

    mgr -> alt : read()
    alt -> data : write altimeter section
    alt --> mgr : true / false

    mgr -> pt  : read()
    pt  -> data : write pt1000 section
    pt  --> mgr : true / false

    mgr -> wd  : read()  [tickClock() + I2C poll]
    wd  -> data : write watchdog section
    wd  --> mgr : true / false

    main -> wd   : sendStatus(status_byte)
    note right : bits: IMU_OK | ALT_OK | PT_OK

    main -> main : printAdcsStatus()

end
@enduml

    6. Recovery Policy¶

    @startuml
start

:module.read();

if (returns false?) then (yes)
    :data ignored\nvalid = false;
    :failCount++;
    if (failCount >= 3?) then (yes)
        :WatchdogComm::sendStatus()\nflags faulty sensor;
    else (no)
    endif
else (no)
    :failCount = 0;
    :g_adcs updated;
endif

stop
@enduml
    Event Immediate reaction After 3 failures
    init() → false module disabled, Serial log stays disabled
    read() → false valid = false watchdog notified via sendStatus()
    Locked I2C bus Wire.end() / Wire.begin() module permanently disabled
