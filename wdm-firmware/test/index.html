    Plan de tests — Watchdog WDM¶

    Contexte¶

    Le Watchdog détecte les fautes sur 5 sous-systèmes via des broches FAULT actives-basses (TPS1H200 open-drain, INPUT_PULLUP sur RP2040). Deux niveaux de test : 1. Tests unitaires — logique testée en isolation avec des mocks logiciels, sans hardware 2. Tests hardware — simulation de fautes sur le vrai Pico par loopback GPIO

    Changements de production requis¶

    1. include/subsystems/PowerChannel.h¶

    Rendre isFault() virtuelle pour permettre l'override par un mock : 

    virtual bool isFault() const;
    Coût : 1 pointeur vtable (4 bytes) × 5 instances = 20 bytes sur 264 KB RAM.

    2. include/behavior/Watchdog.h¶

    Ajouter l'injection de canaux. En production _channels == nullptr → comportement inchangé. 

    #include "subsystems/PowerChannel.h"

// dans la classe :
void setChannels(PowerChannel** channels, uint8_t count);

// private :
PowerChannel** _channels;      // nullptr = production globals
uint8_t        _channelCount;

    3. src/behavior/Watchdog.cpp¶

    init() — initialiser les nouveaux membres : 

    _channels     = nullptr;
_channelCount = 0;

    Ajouter setChannels() : 

    void Watchdog::setChannels(PowerChannel** ch, uint8_t n) {
  _channels = ch; _channelCount = n;
}

    checkAllFaults() — brancher sur _channels : 

    PowerChannel* def[] = { &ibNav, &adcs, &jetson, &ttc, &a7sii };
PowerChannel** ch   = _channels ? _channels : def;
uint8_t        cnt  = _channels ? _channelCount : 5;
for (uint8_t i = 0; i < cnt; i++) { ... }

    MockPowerChannel (test/unit_watchdog/MockPowerChannel.h)¶

    #pragma once
#include "subsystems/PowerChannel.h"

class MockPowerChannel : public PowerChannel {
public:
  explicit MockPowerChannel(const char* name)
    : PowerChannel(0, 0, name), _faultState(false) {}

  void setFault(bool s) { _faultState = s; }
  virtual bool isFault() const override { return _faultState; }

private:
  bool _faultState;
};
    - Les pins 0,0 sont des valeurs fictives — digitalRead/digitalWrite ne sont jamais appelés - Pas d'allocation dynamique, pas de STL

    Pattern setUp : wdt.init(8000) + wdt.setChannels(channels, 3) + reset des 3 mocks

    Tests hardware (test/hw_watchdog/test_main.cpp) — 10 cas¶

    Câblage loopback requis (5 fils)¶

    Canal Broche FAULT Broche driver Action
    IbNav GPIO 5 GPIO 19 driver LOW → isFault() = true
    ADCS GPIO 11 GPIO 25 driver LOW → isFault() = true
    Jetson GPIO 17 GPIO 27 driver LOW → isFault() = true
    TTC GPIO 22 GPIO 28 driver LOW → isFault() = true
    A7Sii GPIO 26 GPIO 29 driver LOW → isFault() = true

    Les broches FAULT TPS1H200 sont open-drain. Quand la puce n'est pas alimentée, la broche flotte et INPUT_PULLUP la tient à HIGH. Le driver RP2040 peut la tirer à LOW sans contention.

    Cas de test¶

    ID Ce qui est vérifié
    TC-HW-01 Baseline : toutes les broches FAULT lisent HIGH (pas de faute)
    TC-HW-02 Loopback IbNav : driver LOW → ibNav.isFault() = true
    TC-HW-03 Loopback ADCS
    TC-HW-04 Loopback Jetson
    TC-HW-05 Loopback TTC
    TC-HW-06 Loopback A7Sii
    TC-HW-07 checkAllFaults() détecte FAULT + CLEAR réels (canal TTC)
    TC-HW-08 5 canaux en faute simultanément → 5 FAULT, puis 5 CLEAR
    TC-HW-09 Pullup tient HIGH quand driver en INPUT (fil débranché simulé)
    TC-HW-10 kick() ne corrompt pas l'état de faute

    Configuration platformio.ini¶

    [env:test_unit]
board          = pico
upload_port    = COM10
test_port      = COM10
test_speed     = 115200
test_framework = unity
test_build_src_filter =
    +<behavior/Watchdog.cpp>
    -<boot/>
    -<subsystems/>
    -<main.cpp>

[env:test_hw]
board          = pico
upload_port    = COM10
test_port      = COM10
test_speed     = 115200
test_framework = unity
test_build_src_filter =
    +<*>
    -<main.cpp>
    • test_unit : exclut les .cpp des sous-systèmes pour éviter les globaux hardware (ibNav, adcs…)
    • test_hw : inclut tout sauf main.cpp (conflit avec le point d'entrée Unity)

    Commandes¶

    pio test -e test_unit        # tests logiciels (pas de câblage nécessaire)
pio test -e test_hw          # tests hardware (câblage loopback requis)
pio test -e test_unit -v     # verbose : voir les [FAULT]/[CLEAR] en live

    Note sur la capture Serial dans Unity¶

    PlatformIO génère automatiquement un unity_config.c qui redirige unityOutputChar() → Serial.write(). Les lignes [FAULT] / [CLEAR] émises par checkAllFaults() apparaissent dans le flux mais sont ignorées par le parser Unity (format incorrect). Elles restent visibles en mode -v. Ne pas appeler Serial.begin() dans les tests — Unity le gère.
