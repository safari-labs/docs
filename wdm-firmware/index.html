    WDM — Watchdog Manager¶

    On-board computer (OBC) firmware for a stratospheric weather-balloon payload, running on a Raspberry Pi RP2040 (dual Cortex-M0+). The WDM supervises power to five subsystems through TPS1H200AQDGNRQ1 smart power switches, detects and recovers from faults with a graded escalation policy, collects telemetry, and downlinks it to ground over an RFD900x radio.

    Status: 81/81 native unit tests passing · production firmware builds for the Pico target.

    Overview¶

    The OBC runs both RP2040 cores in parallel:

    Core Loop Responsibility
    Core 0 loop() Subsystem data collection · telemetry assembly + downlink (1 Hz) · command terminal
    Core 1 loop1() Watchdog: FAULT polling · graded escalation · heartbeat monitoring · WDT kick

    Each subsystem is an independent power channel that can be enabled, disabled, or power-cycled. The watchdog reacts to hardware FAULT signals (TPS1H200 open-drain, active-low) and to loss of activity (CLK heartbeat), applying a progressive recovery strategy before giving up on a channel.

    Peripheral allocation¶

    [IbNav GPS] --I2C0 (GPIO 0/1)--->
[ADCS]      --UART1 (GPIO 8/9)-->  [RP2040 OBC] --packet--> [RFD900x TT&C] --> Ground
[Jetson]    --I2C1 (GPIO 14/15)->               (UART0, GPIO 28/29)
    Bus Assigned to
    I2C0 IbNav (GPS, NMEA over DDC)
    I2C1 Jetson
    UART0 TT&C (RFD900x, 57600 bps)
    UART1 ADCS (raw text stream, 115200 bps)

    Power channels¶

    # Subsystem EN FAULT Output Current limit Role
    0 IbNav GPIO 6 GPIO 5 5 V 2 A GPS / navigation
    1 ADCS GPIO 12 GPIO 11 5 V 1 A IMU + baro + 8× temp
    2 Jetson GPIO 18 GPIO 17 15 V 4 A On-board compute
    3 TT&C GPIO 23 GPIO 22 5 V 3.3 A Ground radio link
    4 A7Sii GPIO 24 GPIO 26 5 V 1.5 A Camera

    TPS1H200 pins: EN (RP2040 → chip, HIGH = ON) and FAULT (chip → RP2040, open-drain, LOW = fault, held HIGH by INPUT_PULLUP).

    Features¶

    • Graded fault escalation — power cycle → extended (non-blocking) disable → permanent disable, driven by a per-channel fault counter.
    • Inter-core safety watchdog — Core 0 proves liveness each loop; if it stalls, Core 1 triggers a full system reset. If both cores stall, the RP2040 hardware WDT reboots the chip.
    • Heartbeat monitoring — CLK toggle detection for ADCS, Jetson and A7Sii; silence beyond timeout is treated as a fault.
    • Telemetry — 124-byte packet, Cortex-M0+ aligned floats, CRC-16/CCITT-FALSE, downlinked at 1 Hz.
    • Command terminal — non-blocking line interpreter over USB serial (bench) or TT&C (in flight).
    • Dependency-injected watchdog — fully unit-testable on a host PC, no hardware required.

    Build & test¶

    Requires PlatformIO.

    # Run all unit tests on the host PC — no hardware needed (81 tests)
pio test -e native

# Build the production firmware for the Pico target
pio run -e pico

# Flash to a connected Pico (set the port in platformio.ini)
pio run -e pico -t upload

    The native environment compiles the pure logic (behavior/ + PowerChannel) against lightweight mocks in test/mocks/, so the full watchdog, telemetry, FSM and terminal logic can be verified without a board.

    Watchdog escalation¶

    The response is driven by a per-channel fault counter, incremented on each rising FAULT edge:

    Fault count Action Constant
    < 5 Power cycle (500 ms off/on) POWER_CYCLE_MS
    ≥ 5 Extended disable, 30 s, non-blocking auto re-enable EXTENDED_DISABLE_MS
    ≥ 10 Permanent disable + ground alert FAULT_PERM_THRESHOLD

    A channel healthy for 60 s (RECOVERY_RESET_MS) has its counter cleared. The extended disable uses a millis() deadline (_disableUntil[ch]) checked in checkAllFaults() — never a blocking delay(), which would starve the 8 s hardware WDT.

    Telemetry packet¶

    124 bytes, transmitted at 1 Hz over the RFD900x in transparent serial mode:

    0   sync[2] = 0xAA 0x55     8   lat,lon (float)      76  temp[8] (float×8)
2   version = 0x01         16   altGPS,speed,heading 108  jetsonStatus,CPU
3   flightState            28   gpsFix,gpsSats       112  jetsonTemp (float)
4   uptime (uint32)        32   altBaro,pressure     116  battVoltage (float)
                           40   accel/gyro (float×6) 120  faultFlags
                           64   roll,pitch,yaw       122  crc16 (CCITT)

    All float fields sit at 4-byte-aligned offsets (Cortex-M0+ has no unaligned access). The CRC-16/CCITT-FALSE covers bytes 0–121.

    Command terminal¶

    Send newline-terminated commands over USB serial or TT&C:

    Command Effect
    help list commands
    status fault bitmask, disabled mask, Core 0 liveness, uptime
    counters per-channel fault count and state
    reset <0-4> clear a channel's fault counter
    disable <0-4> permanently disable a channel
    kick manually feed the WDT
    state current flight phase
    reboot confirm full system reset (confirmation word required)

    Implemented and tested: power channels, boot sequencing + disabled-mask propagation, full watchdog logic, telemetry pipeline (build/CRC/transmit), command terminal.

    Remaining work (see WATCHDOG_REPORT.md §7 for the full table):
